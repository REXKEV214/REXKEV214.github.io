Linux 常用命令清单（附实际案例）

发表于2023-08-19|更新于2025-10-30|运维工具
|浏览量:

整理我日常最常用的命令行工具，每个命令都包含「作用说明 → 常用参数 → 示例」。一篇搞定统计、压缩、文件尾部监控、权限修改、文本搜索与替换。

1. wc：文件统计

参数 说明
-w 统计单词数量
-l 统计行数
-c 统计字节数
-m 统计字符数

示例：cat access.log | wc -l → 统计日志行数。

2. tar：打包与解压

参数 含义
-c 创建归档
-x 解压归档
-z 使用 gzip 压缩（tar.gz）
-v 显示详细过程
-f 指定文件名（必须紧跟在参数后）

3. tail / head：查看文件头尾

  • tail -F：持续监控文件（日志轮转后依旧有效）
  • head -n -5 file：输出到倒数第 5 行为止

4. vim 常用快捷键

模式 快捷键 说明
普通模式 Ctrl+f / Ctrl+b 向前/向后翻页
普通模式 Ctrl+d / Ctrl+u 向下/向上半页
普通模式 yy / p / P 复制当前行 / 粘贴
普通模式 $ / 0 / gg / G 行尾 / 行首 / 文首 / 文末
命令行模式 : set number 显示行号
命令行模式 : /keyword / ?keyword 向下 / 向上搜索

温馨提示：任何命令前都可以加数字（例如 3yy 连续复制 3 行）。

5. chmod：权限设置

符号 意义 数字值
u 所有者 —
g 所有者所在用户组 —
o 其他用户 —
a 所有人 —
r 读权限 4
w 写权限 2
x 执行权限 1

常见操作：

-R 递归、-v 显示详情、-c 仅打印发生变化的条目。

6. grep：文本搜索

参数 作用
-i 忽略大小写
-r 递归目录
-E 支持扩展正则表达式
-w 匹配完整单词
-n 显示行号
-l 只输出文件名
-v 反向匹配（输出不包含的行）

多条件搜索：grep -E 'hello|today' file.txt

7. sed：流编辑器

常用选项：

选项 说明
-n 安静模式，只输出匹配行
-i 就地修改文件
-e 指定脚本
-f 从脚本文件读取命令

核心命令：

命令 功能
a 在行后追加内容
i 在行前插入
d 删除行
c 整行替换
s 字符串替换
p 打印匹配行

示例：

脚本文件（cmd.sed）：

执行：sed -f cmd.sed demo.txt

8. 管道与重定向

  • 管道 |：把前一个命令的标准输出传给下一个命令的标准输入。
  • 输出重定向 > / >>：覆盖 / 追加到文件。
  • 错误输出重定向 2>，同时重定向输出和错误：command >out.log 2>&1。

这些命令几乎覆盖了日常开发、运维的 80% 场景。建议结合 fzf、rg 等工具进一步提升效率，同时把常用组合写成别名或脚本，重复劳动只做一次。*** End Patch
